## ⏳ Việc maintainer làm cho v1.0
1. **Khóa ký registry** (1 lần):
   - `registryctl keygen` → lưu PRIVATE_KEY vào **GitHub secret `REGISTRY_PRIVATE_KEY`**;
     đặt PUBLIC_KEY vào installer env `QUICKWIN_REGISTRY_PUBKEYS` (hoặc default config).
   - KHÔNG commit private key.
2. **CI secret `BANNED_WORDS_REGEX`** (cần token quyền Secrets read-write — hiện read-only).
3. **E2E trên hạ tầng thật** (chưa tự động trong CI):
   - VM Linux sạch (Debian + RHEL): tar.gz → `install.sh` → login → chạy task → reboot 2 service lên.
   - Máy Win11 lab: `winrs-cert` cert auth thật (tests/e2e/winrs/README.md).
4. **Bản dựng UI đầy đủ**: `FULL_UI=1 build-all.sh` (cần node) — CI ui-build đã cover compile;
   release.yml build FULL_UI cho artifact.
5. **Tag ký số** `quickwin-v1.0.0-sem2.18.16` (GPG/sigstore) → push → `release.yml` chạy
   (GitHub Release + publish registry public). **Chỉ maintainer tạo tag.**
6. Đọc lại release note (không lộ thông tin nội bộ) trước publish.
